Google Sheet sync — unit documentation

Auto-saves records as rows in your unit's shared Google Sheet, split into two tabs so personal and department-level data stay separate: Staff_Records (certificates, learning scores, competency assessments, skills checklists, and quick-logged procedures — each row tagged with a "RecordType" column) and ICN_Surveillance (bundle audits, the daily device/skin exception line list, and RCAs). Because this app is a single offline file with no server, it connects via a small Apps Script "bridge" you deploy once on your own sheet — Google does not allow a static webpage to write to Sheets directly without one. The two tabs are created automatically the first time each type of record is synced, and department managers can build pivot tables directly off them.

  1. Open your sheet: NICU unit documentation sheet.
  2. In the sheet, go to Extensions → Apps Script.
  3. Delete any placeholder code and paste in the script below (also copyable with the button).
  4. Click Deploy → New deployment. Choose type Web app. Set "Execute as" to Me and "Who has access" to Anyone. Click Deploy and authorize when prompted.
  5. Copy the Web app URL it gives you, paste it below, and click Save.
  6. Click "Push all data now" once to backfill anything already recorded — new certificates, audits, RCAs, competency checks and quiz attempts will sync automatically from then on.

Note: browsers can't read the response from a cross-origin request, so this app can't confirm success in real time — open the sheet directly to verify rows are appearing after your first sync.
